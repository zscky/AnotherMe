# 拍题生成视频页面修复总结

## 问题
拍题生成视频页面需要刷新一下才能从"题目识别"状态转到"排队等待"状态。

## 根本原因
React 18 的自动批处理 + 异步代码执行的时序问题：
1. `handleGenerate()` 中有多个 `setState` 调用
2. `await sleep(150)` 导致批处理被拆分
3. 轮询开始时，React 的 `useEffect` 还没来得及将新状态保存到 `localStorage`
4. 用户刷新页面时读取的仍是旧的 `localStorage` 数据

## 解决方案
在状态转移的关键点使用 `React.flushSync` 强制同步更新，确保所有状态变化和 localStorage 持久化完成后再开始轮询。

## 修改内容

### 文件修改：`app/(dashboard)/photo-to-video/page.tsx`

#### 1. 添加 flushSync import
```typescript
import { useEffect, useMemo, useRef, useState, flushSync } from 'react';
```

#### 2. 在 handleGenerate 中使用 flushSync
**修改位置**：约第 850-870 行

**修改前**：
```typescript
markStageAsActive('creating_job');
setOverallProgress((prev) => Math.max(prev, 8));
await sleep(150);
markStageAsActive('queueing');
setStatusText('任务已创建，等待处理...');
setOverallProgress((prev) => Math.max(prev, 10));

const pollUrl = createPayload.pollUrl || `/api/problem-video/${createPayload.jobId}`;
const pollIntervalMs = createPayload.pollIntervalMs || 3000;
createdJobMeta = {
  jobId: createPayload.jobId,
  pollUrl,
  pollIntervalMs,
  title: jobTitle,
};
setActiveJob(createdJobMeta);

await pollJobUntilTerminal({
  jobMeta: createdJobMeta,
  controller,
  redirectOnSuccess: true,
});
```

**修改后**：
```typescript
markStageAsActive('creating_job');
setOverallProgress((prev) => Math.max(prev, 8));
await sleep(150);

const pollUrl = createPayload.pollUrl || `/api/problem-video/${createPayload.jobId}`;
const pollIntervalMs = createPayload.pollIntervalMs || 3000;
createdJobMeta = {
  jobId: createPayload.jobId,
  pollUrl,
  pollIntervalMs,
  title: jobTitle,
};

// 关键修复：使用 flushSync 确保状态变化被同步更新并保存到 localStorage
// 避免用户在轮询开始前刷新页面时看到过时的状态
flushSync(() => {
  markStageAsActive('queueing');
  setStatusText('任务已创建，等待处理...');
  setOverallProgress((prev) => Math.max(prev, 10));
  setActiveJob(createdJobMeta);
});

await pollJobUntilTerminal({
  jobMeta: createdJobMeta,
  controller,
  redirectOnSuccess: true,
});
```

## 工作原理

1. **flushSync 的作用**：
   - 强制 React 立即执行所有待处理的 setState 
   - 完成组件的同步渲染
   - 触发 useEffect，将新状态保存到 localStorage
   - 整个过程同步完成，没有异步延迟

2. **修复流程**：
   ```
   标记创建任务阶段 → sleep(150ms) → 
   创建轮询参数 → 
   [flushSync 同步点] →
   标记排队阶段 + 更新所有相关状态 →
   React 立即渲染 →
   useEffect 立即保存 localStorage →
   开始轮询
   ```

3. **结果**：
   - 用户看到的状态始终是一致的
   - 即使在轮询期间刷新，也能看到最新的进度状态
   - localStorage 中的数据始终与页面显示状态同步

## 验证方法

### 功能测试
1. 打开拍题生成视频页面
2. 上传题目图片
3. 点击"生成讲解"

   - 后续的生成步骤...
5. 在任何阶段刷新页面，应该看到实时的最新状态

打开浏览器开发者工具，Application > Local Storage：
1. 查找 `anotherme:dashboard:problem-video-progress:v1` 的值
2. 确保 `backendStepText` 和 `stageStatusMap` 与界面显示状态一致
3. 刷新页面后，localStorage 中的值应该能正确恢复页面状态

## 性能影响
- 最小：`flushSync` 只在一个关键转移点使用
- 强制同步的性能成本可以忽略不计（通常 < 1ms）
- 轮询仍然是异步的，不会阻塞 UI

## 向后兼容性
- 完全兼容：只改变了状态更新的时序，不改变逻辑
- 不影响其他组件：修改局限于该页面
- React 18+ 都支持 `flushSync`
