实施计划
第一阶段：补齐数据模型。

扩展 LiveBookBlockType，加入 callout / figure / flash_cards / code / timeline / concept_graph / user_note / placeholder。
给 LiveBookBlock 增加 paramsJson / metadataJson / error / createdAt / updatedAt。
给 LiveBookChapter 增加 learningObjectives / contentType / sourceRefs / prerequisites / summary。
新增 live_book_explorations 表，存储资料探索结果。
新增 live_book_concept_graphs 或在 live_books 中保存 concept_graph_json。
保持老数据兼容，迁移时默认填空对象。
第二阶段：把模板编译拆成真正编译器。

从 live-book-engine.ts (line 311) 的 compilePageBlocks 中拆出 LiveBookCompiler。
当前模板逻辑保留为 fallback。
新增 BlockGeneratorRegistry，每种区块一个 generator。
编译时按区块逐个保存，并发送 block_ready / block_error / page_ready 事件。
失败区块不阻塞整页，页面状态支持 partial。
第三阶段：实现资料探索。

新增 lib/server/live-book/source-explorer.ts。
输入来源先做最小闭环：topic + current classroom scenes + notebook records + quiz attempts。
后续再接 PDF/RAG。
输出结构对齐 DeepTutor 的 ExplorationReport：queries / chunks / summary / coverage / candidateConcepts。
编译区块时优先读取 exploration，避免每个区块重复检索。
第四阶段：重做书脊生成。

当前 planSpine(topic) 是固定 4 章模板，应改为：
先生成 proposal。
用户确认后运行 SourceExplorer。
再运行 SpineSynthesizer 生成章节、学习目标、概念图、先修关系。
支持用户在 UI 中调整章节顺序后再确认。
自动插入“本书导览”页，展示概念图和章节入口，借鉴 DeepTutor 的 overview chapter 思路。
第五阶段：页面规划和区块类型升级。

新增 SectionArchitect，根据 contentType 规划页面区块。
推荐模板：
theory：section + figure + callout + quiz + flash_cards
derivation：section + animation + code + quiz
practice：section + quiz + interactive + remedial
concept：section + concept_graph + example + quiz
当前页面 renderer 只渲染纯文本，应改为按 block type 分组件渲染。
现有 Stage 适配继续保留，但要传递 payload，而不是只传 content 字符串。
第六阶段：交互和个性化。

页内追问不再只追加固定 deep_dive 文本，而是调用 deepDiveBlockGenerator。
答错后不只追加固定 remedial 文案，而是基于错题、章节目标、source anchors 生成补救讲解和低难度练习。
将 quizAttempts 反写到 progress，更新 weakChapterIds 和推荐复习路径。
后续可接 anotherme2_engine/agents/foundation/learning_block.py 里的学习块模型。
第七阶段：前端体验改造。

app/(dashboard)/live-book/page.tsx 保留工作台，但拆成组件：书库、创建器、目录编辑器、阅读器、进度时间线、洞察面板。
LiveBookRenderer 改成真正 block renderer：
quiz 显示选项和判题。
figure 支持 Mermaid/SVG/Chart。
interactive 使用 sandbox iframe。
animation 对接现有视频/Manim 结果。
concept_graph 显示概念依赖。
舞台页 page.tsx (line 35) 继续复用 liveBookToStage，但要把活书区块转成可播放 scene。
优先级建议
MVP 先做 4 件事：

数据模型扩展：block metadata、source refs、concept graph。
编译器拆分：从模板函数变成 compiler + generators。
SourceExplorer 最小版：从主题、当前课堂、笔记、错题收集 evidence。
BlockRenderer 升级：至少支持 section / quiz / callout / concept_graph / deep_dive / remedial。
后续再做：

PDF/RAG 接入。
代码、图表、交互 HTML、动画区块。
知识库漂移检测。
更完整的学习画像和复习路径。
时间排期
建议按 4 周推进：

第 1 周：模型与存储
完成 DB migration、类型扩展、兼容旧数据、区块 payload/schema 约定。

第 2 周：编译器核心
拆出 LiveBookCompiler、BlockGeneratorRegistry、区块级事件、失败恢复、单块重生成。

第 3 周：资料驱动生成
实现 SourceExplorer、SpineSynthesizer、概念图、overview 页，替换固定目录模板。

第 4 周：体验闭环
完善 block renderer、页内追问、答错补救、学习洞察、Stage 播放适配。

主要风险
最大风险不是技术迁移，而是生成质量不可控。建议所有 LLM 生成都必须有：

JSON schema 校验。
fallback 模板。
区块级失败隔离。
source anchors。
编译日志和事件时间线。
单页、单块重试入口。
这样即使某个区块失败，整本活书仍可阅读、可修复、可继续编译。
